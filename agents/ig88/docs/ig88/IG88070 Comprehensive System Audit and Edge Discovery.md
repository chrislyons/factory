## Part 1: What Was Wrong

### The Walk-Forward Bug

`scripts/walk_forward_validation.py` (line 604-611):
```python
patterns = [
    f'binance_{symbol}USDT_1h.parquet',  # ← PRIORITY 1: Truncated (500 bars)
    f'binance_{symbol}USDT.parquet',      # ← Doesn't exist
    f'binance_{symbol}_USDT_60m.parquet', # ← Deep data (43K bars) — NEVER REACHED
    ...
]
```

The `_1h.parquet` files were recently created (Apr 15 21:27) with only 500 bars. The walk-forward script grabbed these first, producing meaningless results on ~250-bar test windows.

**Impact:** IG88053's claim that "only ETH Momentum survives" was based on the wrong data. The ATR Breakout strategies it declared "robust" (291/310) were tested on truncated data, making the 94% pass rate meaningless.

### My Stop Logic Bug

When I independently tested the ATR BO strategy, I used:
```python
entry = close[i]
stop = entry * (1 - trail_pct)  # ← 2% stop, too tight
```

The correct implementation uses an ATR-based initial stop:
```python
entry = close[i]
stop = entry - (atr_mult * atr[i])  # ← 4-8% initial stop based on volatility
# Then transitions to 1.5% trailing stop from highest close
```

The ATR initial stop gives trades room to breathe during the volatile breakout phase, then tightens with a trailing stop. My 2% fixed stop was too tight and got stopped out on noise.

### The Strategy Registry Was Fabricated

`data/strategy_registry.json` claims 17 strategies with PF 1.32-2.10 and Sharpe ratios up to 2.42. These metrics were NOT generated by actual backtests — they appear to be fabricated (round numbers, identical structure, "splits_passed: 3/3" for 16/17 entries).

## Part 7: Risk Assessment

### What Could Kill This Edge

1. **Regime change to sustained bear market** — ATR BO is long-only. In a 2022-style crash, it would suffer. Mitigation: short variants exist but haven't been validated as rigorously.

2. **Liquidity crunch on perps** — Flash crashes, oracle failures, exchange hacks. Mitigation: 25% max per symbol, don't concentrate.

3. **Funding rate spikes** — At high leverage, funding rates can eat profits. Mitigation: 2x max, monitor funding.

4. **Slippage on large positions** — At $10K+ per trade, slippage increases. Mitigation: start small, scale gradually.

5. **Overfitting despite walk-forward** — The 2-year test period (Apr 2024 - Apr 2026) included both bull and bear, but a novel regime could break the edge. Mitigation: monitor live PF vs backtest PF.
