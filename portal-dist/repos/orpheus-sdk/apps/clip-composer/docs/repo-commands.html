Clip Composer - Command Reference

Quick reference for Clip Composer development tasks

Last updated: 2026-02-01

Project Path
~/dev/orpheus-sdk/apps/clip-composer — JUCE 8.0.4 + Orpheus SDK, C++20, CMake build system
Build Prerequisite
The Orpheus SDK must be built first from the repo root (~/dev/orpheus-sdk) before building Clip Composer. All cmake commands below should be run from the SDK root unless noted otherwise.
Job Command Description
Build
Configure CMake (Debug) cd ~/dev/orpheus-sdk && cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug Configure SDK + OCC build system in Debug mode
Configure CMake (Release) cd ~/dev/orpheus-sdk && cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release Configure SDK + OCC build system in Release mode (optimized)
Build OCC (Debug) cd ~/dev/orpheus-sdk && cmake --build build --target orpheus_clip_composer_app Build only the Clip Composer application target (Debug)
Build OCC (Release) cd ~/dev/orpheus-sdk && cmake --build build-release --target orpheus_clip_composer_app Build optimized Release version of Clip Composer
Build OCC (parallel) cd ~/dev/orpheus-sdk && cmake --build build --target orpheus_clip_composer_app -j$(sysctl -n hw.ncpu) Build OCC using all CPU cores for faster compilation
Build entire SDK + OCC cd ~/dev/orpheus-sdk && cmake --build build Build all SDK targets including Clip Composer
Clean rebuild OCC only cd ~/dev/orpheus-sdk && rm -rf build/apps/clip-composer && cmake --build build --target orpheus_clip_composer_app Clean rebuild only OCC (preserves SDK build artifacts)
Full clean rebuild cd ~/dev/orpheus-sdk && rm -rf build && cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug && cmake --build build Nuclear option: full clean rebuild of SDK + OCC (use when build is corrupted)
Run
Relaunch OCC (recommended) cd ~/dev/orpheus-sdk && ./scripts/relaunch-occ.sh Clear cache, rebuild, and relaunch Clip Composer (recommended development workflow)
Launch OCC (Debug) open ~/dev/orpheus-sdk/build/apps/clip-composer/orpheus_clip_composer_app_artefacts/Debug/OrpheusClipComposer.app Launch Clip Composer Debug build directly (macOS)
Launch OCC (Release) open ~/dev/orpheus-sdk/build-release/apps/clip-composer/orpheus_clip_composer_app_artefacts/Release/OrpheusClipComposer.app Launch Clip Composer Release build (macOS)
Quick rebuild + launch cd ~/dev/orpheus-sdk && cmake --build build --target orpheus_clip_composer_app && open build/apps/clip-composer/orpheus_clip_composer_app_artefacts/Debug/OrpheusClipComposer.app Fast rebuild and launch (no cache clear)
View OCC logs tail -f ~/Library/Logs/OrpheusClipComposer/*.log Monitor Clip Composer application logs in real-time
Clear OCC preferences rm -rf ~/Library/Application\ Support/OrpheusClipComposer/ Reset OCC to default settings (removes sessions and user data)
Testing
Run all OCC tests cd ~/dev/orpheus-sdk && ctest --test-dir build -R ClipComposer --output-on-failure Run all Clip Composer-specific tests
Run OCC integration tests cd ~/dev/orpheus-sdk && ctest --test-dir build -R OCC_Integration --output-on-failure Run OCC integration tests with Orpheus SDK
Run all SDK + OCC tests cd ~/dev/orpheus-sdk && ctest --test-dir build --output-on-failure Run complete test suite (SDK + OCC)
Run tests (verbose) cd ~/dev/orpheus-sdk && ctest --test-dir build -R ClipComposer --output-on-failure --verbose Run OCC tests with detailed output for debugging
Run specific test cd ~/dev/orpheus-sdk && ctest --test-dir build -R TestName --output-on-failure Run only tests matching TestName pattern
Run tests (parallel) cd ~/dev/orpheus-sdk && ctest --test-dir build -R ClipComposer --output-on-failure -j4 Run OCC tests in parallel (4 jobs)
Build + test (one-shot) cd ~/dev/orpheus-sdk && cmake --build build --target orpheus_clip_composer_app && ctest --test-dir build -R ClipComposer --output-on-failure Build OCC then immediately run its tests
