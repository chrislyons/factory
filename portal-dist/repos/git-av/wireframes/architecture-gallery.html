Architecture Overview

System Layers

High-level system design from CLI through core engine to storage, VCS, and C2PA provenance layers. The Trust Ladder provides progressive hashing (inode → metadata → sparse → full) for large media files. Symlink Swizzle enables sub-second branch checkout.

Data Flow

Operation Sequences

Sequence diagrams for the three core operations: Snapshot (hash → store → symlink → record → manifest), Checkout (state lookup → diff → batch symlink swizzle → HEAD update), and Revert (history lookup → atomic symlink replace). Includes C2PA auto-manifest generation with optional git context injection.

Database Schema

SQLite v3 - 7 Tables

SQLite ledger schema evolved across 3 progressive migrations: v1 (blobs + versions), v2 (branches + head_pointer + versions.branch_id), v3 (c2pa_manifests + c2pa_actions + c2pa_ingredients). All migrations are idempotent and run automatically on startup.

erDiagram blobs { TEXT hash PK "BLAKE2b content hash" TEXT original_path "First-seen file path" TEXT created_at "ISO 8601 timestamp" } versions { INTEGER id PK "AUTOINCREMENT" TEXT file_path "Workspace file path" TEXT blob_hash FK "References blobs.hash" TEXT description "User-provided message" TEXT created_at "ISO 8601 timestamp" INTEGER branch_id FK "References branches.id (v2)" } branches { INTEGER id PK "AUTOINCREMENT" TEXT name UK "Unique branch name" TEXT created_at "ISO 8601 timestamp" } head_pointer { INTEGER singleton PK "CHECK singleton = 1" INTEGER current_branch_id FK "References branches.id" } c2pa_manifests { TEXT blob_hash PK "References blobs.hash" TEXT manifest_path "Sidecar .c2pa.json path" TEXT manifest_hash "Hash of manifest content" INTEGER signature_valid "Boolean 0 or 1" TEXT signed_at "ISO 8601 timestamp" } c2pa_actions { INTEGER id PK "AUTOINCREMENT" TEXT blob_hash FK "References blobs.hash" TEXT action_type "c2pa.created, c2pa.edited" TEXT software_agent "Tool or application" TEXT digital_source_type "AI source classification" TEXT parameters "JSON-encoded params" TEXT timestamp "ISO 8601 timestamp" } c2pa_ingredients { TEXT parent_hash PK "References blobs.hash" TEXT ingredient_hash PK "References blobs.hash" TEXT relationship "parentOf, componentOf" } blobs ||--o{ versions : "content identity" blobs ||--o| c2pa_manifests : "provenance sidecar" blobs ||--o{ c2pa_actions : "action history" blobs ||--o{ c2pa_ingredients : "parent content" blobs ||--o{ c2pa_ingredients : "ingredient content" branches ||--o{ versions : "branch context" branches ||--o| head_pointer : "current branch"
