Data Flow

Sequence

GraphQL request/response and ingestion cycles

sequenceDiagram participant U as User participant R as React App participant AC as Apollo Client participant W as CF Worker participant D as D1 Database Note over U,D: Query Flow (Grid View) U->>R: Navigate to Grid tab R->>AC: useQuery(GET_EVENTS) AC->>W: POST /graphql W->>D: SELECT events JOIN venues D-->>W: Row results W-->>AC: GraphQL response AC-->>R: Return data R-->>U: Render Grid Note over U,D: Mutation Flow (Authenticated or Anonymous) U->>R: Submit event form R->>AC: useMutation(ADD_EVENT) AC->>W: POST /graphql (JWT optional) W->>W: Check IP rate limit (10/hr) alt Authenticated User W->>D: INSERT events (status=scheduled) W->>D: INSERT moderation_queue (approved) else Anonymous User W->>D: INSERT events (status=pending_approval) W->>D: INSERT moderation_queue (pending) end D-->>W: event_id W-->>AC: Event created AC-->>R: Success response R-->>U: Show confirmation Note over U,D: Ingestion Flow (Cross-Source Dedup) participant CLI as CLI Scripts CLI->>W: ingest-*.js or sync-feeds.js W->>D: Phase 1: Hash lookup event_sources alt Hash match W->>D: Log to skipped_duplicates else No hash match W->>D: Phase 2: Fuzzy match (title/date) alt Match found (aggregator) W->>D: Log to skipped_duplicates else New event W->>D: INSERT event + venue W->>D: Record in event_sources end end Note over U,D: Auth Flow (Magic Link) U->>R: Enter email R->>AC: requestMagicLink(email) AC->>W: POST /graphql W->>D: INSERT magic_link token D-->>W: Token created W-->>AC: Success AC-->>R: Show check email U->>R: Click magic link R->>AC: consumeMagicLink(token) AC->>W: POST /graphql W->>D: Validate and consume token D-->>W: User data W-->>AC: JWT + user AC-->>R: Authenticated R-->>U: Logged in state
