🗄️ Database (D1) - Production: osd-events | Test: osd-events-test
Backup production to R2 ./scripts/backup-db-to-r2.sh Export production database to R2 bucket (ALWAYS run before migrations)
Backup (TypeScript API) export D1_REST_API_TOKEN="your-token" && npx tsx scripts/backup-d1-to-r2.ts pre-migration Advanced backup using REST API (saves to ./backups/ with custom reason)
List R2 backups wrangler r2 object list osd-backups View all available database backups in R2
Restore from R2 backup wrangler r2 object get osd-backups/osd-db-backup-TIMESTAMP.sql --file=restore.sql && wrangler d1 execute osd-events --file=restore.sql ⚠️ OVERWRITES production database - test on osd-events-test first!
Query production DB wrangler d1 execute osd-events --command="SELECT COUNT(*) FROM events;" Run SQL query on production database (osd-events)
Query test DB wrangler d1 execute osd-events-test --command="SELECT COUNT(*) FROM events;" Run SQL query on test database (osd-events-test)
Apply migration (production) wrangler d1 execute osd-events --file=migrations/0020_example.sql Apply migration to PRODUCTION (always backup first!)
Apply migration (test) wrangler d1 execute osd-events-test --file=migrations/0020_example.sql Apply migration to TEST database (safe for testing)
List all tables wrangler d1 execute osd-events --command="SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;" View all database tables
Check table schema wrangler d1 execute osd-events --command="PRAGMA table_info(events);" View columns and types for a specific table
Export database (local file) wrangler d1 export osd-events --remote --output=osd-events-backup.sql Export production database to local SQL file
Database info wrangler d1 info osd-events Show database size, row counts, and storage usage

🔐 SQL: Admin & Moderation Functions
Create first admin wrangler d1 execute osd-events --remote --command "UPDATE agents SET roles = '[\"user\", \"admin:full\"]' WHERE email = 'your@email.com';" Bootstrap first admin user by email (SQL version)
List all admin roles wrangler d1 execute osd-events --remote --command "SELECT agent_id, email, roles FROM agents WHERE roles LIKE '%admin%';" Find all users with admin or moderator roles
Assign moderator by ID wrangler d1 execute osd-events --remote --command "INSERT INTO moderators (moderator_id, agent_id, cities, categories, max_queue_size) VALUES (hex(randomblob(16)), 'AGENT_ID', '[\"Toronto\", \"Montreal\"]', NULL, 50); UPDATE agents SET roles = json_array_insert(COALESCE(roles, '[]'), '$[0]', 'moderator') WHERE agent_id = 'AGENT_ID';" Assign moderator role with city scope (replace AGENT_ID)
View moderation queue status wrangler d1 execute osd-events --remote --command "SELECT status, COUNT(*) as count FROM moderation_queue GROUP BY status;" Count pending/approved/rejected events in queue
View queue age distribution wrangler d1 execute osd-events --remote --command "SELECT CASE WHEN (julianday('now') - julianday(submitted_at)) < (1.0/24) THEN 'Under 1h' WHEN (julianday('now') - julianday(submitted_at)) < (4.0/24) THEN '1-4h' WHEN (julianday('now') - julianday(submitted_at)) < 1 THEN '4-24h' ELSE '24h+' END as age_bucket, COUNT(*) as count FROM moderation_queue WHERE status = 'pending' GROUP BY age_bucket;" Show pending events by age (helps identify SLA breaches)
View audit trail for event wrangler d1 execute osd-events --remote --command "SELECT timestamp, action, agent_id, details FROM audit_log WHERE resource_id = 'EVENT_ID' ORDER BY timestamp DESC;" View all admin actions for a specific event (replace EVENT_ID)
Moderator performance stats wrangler d1 execute osd-events --remote --command "SELECT m.agent_id, a.email, COUNT(d.decision_id) as decisions, SUM(CASE WHEN d.decision = 'approved' THEN 1 ELSE 0 END) as approved FROM moderators m LEFT JOIN agents a ON m.agent_id = a.agent_id LEFT JOIN moderation_decisions d ON m.agent_id = d.moderator_id GROUP BY m.agent_id ORDER BY decisions DESC;" Show each moderator's approval/rejection counts
Check SLA breaches wrangler d1 execute osd-events --remote --command "SELECT queue_id, event_id, submitted_at, sla_deadline, status FROM moderation_queue WHERE status = 'pending' AND sla_deadline < datetime('now');" Find pending events that missed SLA deadline
Rule effectiveness metrics wrangler d1 execute osd-events --remote --command "SELECT rule_id, COUNT(*) as executions, SUM(CASE WHEN result = 'pass' THEN 1 ELSE 0 END) as passes, SUM(CASE WHEN result = 'fail' THEN 1 ELSE 0 END) as failures, ROUND(AVG(execution_time_ms), 2) as avg_ms FROM rule_execution_log GROUP BY rule_id ORDER BY executions DESC;" Show rule execution stats and effectiveness rates
Clear completed queue items wrangler d1 execute osd-events --remote --command "DELETE FROM moderation_queue WHERE status != 'pending' AND reviewed_at < datetime('now', '-90 days');" Archive completed (non-pending) items older than 90 days

Apply ingestion migrations wrangler d1 execute osd-events --remote --file=migrations/0018_add_event_ingestion_schema.sql && wrangler d1 execute osd-events --remote --file=migrations/0019_add_ingestion_moderation_rules.sql Apply event ingestion schema (event_sources, venue_sources, duplicates) and moderation rules to production
Check ingestion tables wrangler d1 execute osd-events --remote --command "SELECT name FROM sqlite_master WHERE type='table' AND name LIKE '%source%' OR name LIKE '%duplicate%';" Verify event_sources, venue_sources, and duplicate tables exist
Check ingestion rules wrangler d1 execute osd-events --remote --command "SELECT rule_id, rule_name, enabled FROM moderation_rules WHERE rule_name LIKE '%trusted%' OR rule_name LIKE '%duplicate%';" View ingestion-related moderation rules (trusted_sources, duplicate_detection, etc.)
Test ingestion mutation (Ticketmaster) curl -X POST https://osd-events.chrislyons.workers.dev/graphql -H "Content-Type: application/json" -H "Authorization: Bearer ADMIN_TOKEN" -d '{"query": "mutation { ingestEventsFromSource(source: \"ticketmaster\", events: [{event_name: \"Test Event\", event_date: \"2025-12-25T18:00:00Z\", city: \"Toronto\", venue: \"Venue Name\", source_event_id: \"tm_test_001\"}]) { success ingested queued stats { autoApprovedCount queuedCount } } }"}' Test Ticketmaster ingestion mutation (requires admin token)
View ingested events wrangler d1 execute osd-events --remote --command "SELECT e.event_id, e.event_name, es.source_name, es.source_event_id FROM events e JOIN event_sources es ON e.event_id = es.event_id LIMIT 20;" View recently ingested events and their source mappings
Check dedup detection wrangler d1 execute osd-events --remote --command "SELECT * FROM event_duplicates LIMIT 20;" View detected event duplicates from ingestion
Swap to test database Edit wrangler.toml: Change d1_database_id to test DB ID (97a11ffc-2aea-4d31-bbb5-6b505651b2e0) && cd apps/worker && wrangler deploy --env test Switch Worker to test database for ingestion testing (must apply migrations 0001-0019 first)
Swap back to production Edit wrangler.toml: Revert d1_database_id to production DB ID && cd apps/worker && wrangler deploy Restore Worker to production database
View ingestion queue status wrangler d1 execute osd-events --remote --command "SELECT source_name, COUNT(*) as ingested_count FROM event_sources GROUP BY source_name;" Show count of events ingested per source (Ticketmaster, Toronto, Edmonton, Quebec SIT)
Venue dedup analysis wrangler d1 execute osd-events --remote --command "SELECT vs.source_venue_id, vs.venue_id, v.venue_name FROM venue_sources vs JOIN venues v ON vs.venue_id = v.venue_id LIMIT 20;" Check venue deduplication mappings (matched existing venues vs newly created)

⚙️ SQL: Database Maintenance
Vacuum & optimize wrangler d1 execute osd-events --remote --command "VACUUM; ANALYZE;" Defragment database and update statistics (improves query performance)
Check database integrity wrangler d1 execute osd-events --remote --command "PRAGMA integrity_check;" Verify database file is not corrupted
List all indexes wrangler d1 execute osd-events --remote --command "SELECT name, tbl_name, sql FROM sqlite_master WHERE type='index' AND tbl_name NOT LIKE 'sqlite_%';" View all table indexes and their definitions
Show database size info wrangler d1 execute osd-events --remote --command "SELECT name, SUM(pgsize) as size_bytes FROM (SELECT name, pgsize FROM pragma_page_count, pragma_page_size, sqlite_master WHERE type='table') GROUP BY name ORDER BY size_bytes DESC;" View storage used by each table (largest first)
Row count per table wrangler d1 execute osd-events --remote --command "SELECT name, (SELECT COUNT(*) FROM sqlite_master m WHERE m.type='table' AND m.name=t.name) as rows FROM (SELECT DISTINCT name FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%') t;" Count total rows in each table
Archive old audit logs wrangler d1 execute osd-events --remote --command "DELETE FROM audit_log WHERE timestamp < datetime('now', '-1 year');" Remove audit logs older than 1 year (compliance: keep >1 year)
Clean stale rate limits wrangler d1 execute osd-events --remote --command "DELETE FROM rate_limit_tracking WHERE window_end < datetime('now', '-1 hour');" Remove expired rate limit windows (cleanup old entries)
Check foreign key constraints wrangler d1 execute osd-events --remote --command "PRAGMA foreign_keys=ON; PRAGMA foreign_key_list(moderation_queue);" Verify foreign key relationships for a table
View slow query candidates wrangler d1 execute osd-events --remote --command "SELECT rule_id, COUNT(*) as executions, AVG(execution_time_ms) as avg_ms, MAX(execution_time_ms) as max_ms FROM rule_execution_log GROUP BY rule_id HAVING avg_ms > 100 ORDER BY avg_ms DESC;" Find rules with slow execution times (>100ms average)
