ksi-lisims Command Reference

Project: ksi-lisims - Research repository

Type: Academic research with IEEE citation style

Prefix: KSI### (e.g., KSI001 Topic Title.md)

Template: Based on undone (historical research pattern)

Convention: This is a research repository. Most commands focus on documentation management, research workflows, and quality assurance rather than code compilation.

📚 Documentation Management

Finding Document Numbers

ls -1 docs/ksi/ | grep -E '^KSI[0-9]{3,4}\s+' | sort
List all KSI-prefixed documentation files
ls -1 canon/v1/ | grep -E '^KSI[0-9]{3,4}' | sort
List canonical version 1 documents
last=$(ls -1 docs/ksi/ | grep -E '^KSI[0-9]+' | sed -E 's/KSI([0-9]+).*/\1/' | sort -n | tail -1); next=$((last + 1)); echo "Next: KSI${next} Your Title Here.md"
Find the next available KSI document number
CRITICAL: Always include a descriptive title after the number. Example: KSI003 Environmental Policy Framework.md NOT KSI003.md

Viewing Documents

cat docs/ksi/KSI001*.md | head -50
Preview first 50 lines of KSI001 document
grep -n "^## " docs/ksi/KSI001*.md
Show section structure with line numbers
wc -w docs/ksi/KSI*.md
Get word counts for all KSI documents

🔍 Research & Citations

Citation Management

grep -E '\[[0-9]+\]' docs/ksi/KSI001*.md | wc -l
Count inline citations in a document
grep -A 1 "^## References" docs/ksi/KSI001*.md
Extract References section from a document
grep -n "http" docs/ksi/KSI001*.md
Find all URLs in a document (for citation verification)
Best Practice: Use the citation-verifier skill to automatically verify all IEEE citations match sources.

Search Content

grep -i "environmental law" docs/ksi/*.md canon/v1/*.md
Search for a term across all research documents
grep -i "analytic note" docs/ksi/KSI001*.md
Find all "Analytic note:" sections (thesis connections)

✍️ Writing Workflow

Word Count Tracking

wc -w docs/ksi/KSI*.md | tail -1
Total word count for all working documents
wc -w canon/v1/KSI*.md | tail -1
Total word count for canonical documents
for f in docs/ksi/KSI*.md; do echo "$f: $(wc -w < "$f") words"; done
Word count for each document individually

Quality Checks

grep -i "inevitable\|destiny\|meant to\|had to" docs/ksi/KSI001*.md
Check for teleological language (retrospective bias)
grep "^## " docs/ksi/KSI001*.md
Review section structure (cursory reading path)
diff docs/ksi/KSI001*.md canon/v1/KSI001*.md
Compare working version to canonical version

📝 Common Research Workflows

Starting a New Chapter

1. Find next number: ls -1 docs/ksi/ | grep KSI | sort
2. Create file: docs/ksi/KSI00X Title.md
3. Use template from CLAUDE.md
4. Research with historical-researcher agent
5. Write with dual-reader design in mind
6. Verify citations with citation-verifier skill
7. Check quality with dual-reader-check skill

Completing a Chapter

1. Run verification-auditor agent for comprehensive review
2. Use academic-critic agent for adversarial review
3. Verify all citations: citation-verifier skill
4. Test readability: dual-reader-check skill
5. Move to canon: mv docs/ksi/KSI00X*.md canon/v1/
6. Update COMPLETION_SUMMARY.md
7. Document in .claude/implementation_progress.md
