🗄️ Database (SQLite)
Start with custom DB path STORAGE_BACKEND=sqlite DB_PATH=./data/ondina.db pnpm start Start server with specific SQLite database file
View database sqlite3 ./data/ondina.db ".tables" List all tables in the database
Query time blocks sqlite3 ./data/ondina.db "SELECT id, title, date(start_time) as date FROM time_blocks LIMIT 10;" View recent time blocks
Check schema version sqlite3 ./data/ondina.db "SELECT * FROM schema_migrations ORDER BY version DESC LIMIT 5;" View applied migrations
Backup database sqlite3 ./data/ondina.db ".backup ./data/ondina-backup.db" Create a backup of the database (safe while server running with WAL)
View table schema sqlite3 ./data/ondina.db "PRAGMA table_info(time_blocks);" Show columns and types for time_blocks table
Count records sqlite3 ./data/ondina.db "SELECT 'time_blocks', COUNT(*) FROM time_blocks UNION SELECT 'daily_reflections', COUNT(*) FROM daily_reflections UNION SELECT 'feedback_entries', COUNT(*) FROM feedback_entries;" Count records in key tables
Check WAL mode sqlite3 ./data/ondina.db "PRAGMA journal_mode;" Verify WAL mode is enabled (should return "wal")

🌐 API Usage
Get policy curl http://localhost:3000/api/policy Retrieve current user policy settings
Update policy curl -X PUT http://localhost:3000/api/policy -H "Content-Type: application/json" -d '{"guardrails":{"no_after_hour":"21:00"},"energy_windows":{"am":"high","mid":"medium","pm":"low"},"churn_penalty":0.35,"deepwork_bias":0.6}' Update policy settings
Submit AM reflection curl -X POST http://localhost:3000/api/reflection -H "Content-Type: application/json" -d '{"serial":"OND20260125","date":"2026-01-25","am":{"sleep_hours":7.5,"sleep_quality":"good","energy":"high","focus":"medium","mood":"optimistic"}}' Submit morning reflection
Get reflection curl http://localhost:3000/api/reflection/2025-01-20 Get reflection for specific date
Generate plan curl -X POST http://localhost:3000/api/plan -H "Content-Type: application/json" -d '{"date":"2025-01-20"}' Generate daily plan for date
Execute plan curl -X POST http://localhost:3000/api/plan/2025-01-20/execute -H "Content-Type: application/json" -d '{"confirmed":true,"rejectedActions":[]}' Execute a generated plan
Import tasks curl -X POST http://localhost:3000/api/ingest/tasks -H "Content-Type: application/json" -d '{"tasks":[{"title":"Write proposal","priority":"high","duration_min":90,"tags":["deepwork"]}]}' Import personal tasks as time blocks
Import work schedule curl -X POST http://localhost:3000/api/ingest/schedule -H "Content-Type: application/json" -d '{"csv":"date,start,end,title\n2025-01-20,09:00,17:00,Work","timezone":"America/New_York"}' Import work schedule from CSV

🔧 Troubleshooting
Port already in use PORT=3001 pnpm start Start on different port if 3000 is occupied
Database locked lsof +D ./data/ | grep ondina Find processes using the database (kill other servers)
TypeScript errors pnpm type-check Run type check to see detailed TS errors
Clean reinstall rm -rf node_modules dist && pnpm install && pnpm build Full clean and rebuild (fixes corrupt dependencies)
Reset database rm ./data/ondina.db* && STORAGE_BACKEND=sqlite pnpm start Delete database and start fresh (migrations run automatically)
Check Node version node --version Must be v22.x or higher
View server logs pnpm start 2>&1 | tee server.log Start server and capture logs to file
