Build Commands

Initial Setup (First Build)

cd ~/dev/freqfinder
rm -rf build
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build . --config Release -j8
Note: First build takes 10-15 minutes as JUCE downloads and compiles. Subsequent builds are much faster (<1 minute).

Clean Rebuild

cd ~/dev/freqfinder
rm -rf build
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build . --config Release -j8

Incremental Build (after code changes)

cd ~/dev/freqfinder/build
cmake --build . --config Release -j8

Debug Build

cd ~/dev/freqfinder
rm -rf build
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build . --config Debug -j8

Installation

Install VST3 Plugin

cp -r ~/dev/freqfinder/build/FreqFinder_artefacts/Release/VST3/Freqfinder.vst3 \
   ~/Library/Audio/Plug-Ins/VST3/

Install AU Plugin

cp -r ~/dev/freqfinder/build/FreqFinder_artefacts/Release/AU/Freqfinder.component \
   ~/Library/Audio/Plug-Ins/Components/

Uninstall Plugins

rm -rf ~/Library/Audio/Plug-Ins/VST3/Freqfinder.vst3
rm -rf ~/Library/Audio/Plug-Ins/Components/Freqfinder.component

Common Issues

Build Fails with "JuceHeader.h not found"

Solution: JUCE 8.x doesn't use JuceHeader.h. Use individual module headers instead: 
#include <juce_audio_processors/juce_audio_processors.h>
#include <juce_dsp/juce_dsp.h>

App Crashes on Launch

Solution: Check crash logs and run from terminal to see console output: 
~/dev/freqfinder/build/FreqFinder_artefacts/Release/Standalone/Freqfinder.app/Contents/MacOS/Freqfinder

CMake Error: CMAKE_C_COMPILE_OBJECT not set

Solution: Ensure project declares both C and CXX languages: 
project(FreqFinder VERSION 0.2.1 LANGUAGES C CXX)

Quick Tests

Verify All Oscillators Work

  1. Launch standalone app
  2. Click frequency display to start fundamental (should hear tone)
  3. Click "2" in Even column (should hear octave up)
  4. Click "3" in Odd column (should hear perfect fifth)
  5. Verify VU meters show levels for active oscillators

Test MIDI Input

  1. Connect MIDI keyboard
  2. Launch standalone app
  3. Settings → Audio/MIDI Settings → Enable MIDI input device
  4. Play MIDI note (frequency should change automatically)

Test Parameter Automation (DAW)

  1. Load VST3 in DAW (Logic, Ableton, Reaper, etc.)
  2. Automate "Master Volume" parameter
  3. Play timeline - volume should change smoothly
  4. No clicks or pops should be audible
