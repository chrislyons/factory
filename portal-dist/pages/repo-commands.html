Matrix Coordinator
View logs ssh blackbox 'journalctl -u matrix-coordinator -f' Stream coordinator logs in real-time
Restart coordinator ssh blackbox 'sudo systemctl restart matrix-coordinator' Restart the Matrix coordinator service
Reload config (no restart) ssh blackbox 'kill -HUP $(pidof node /home/nesbitt/factory/coordinator/src/dist/matrix-coordinator.js)' Hot-reload agent-config.yaml without restarting sessions
Check status ssh blackbox 'systemctl status matrix-coordinator' Show coordinator service status and recent log lines
Check recent errors ssh blackbox 'journalctl -u matrix-coordinator --since "5 min ago" --no-pager | grep -i error' Show errors from the last 5 minutes

LiteLLM Proxy — Greybox (GYX004)
Service status ssh greybox 'systemctl status litellm --no-pager' Check LiteLLM proxy service status (systemd, always-on)
Restart LiteLLM ssh greybox 'sudo systemctl restart litellm' Restart LiteLLM proxy (config: ~/.config/litellm/config.yaml)
Health check curl -s http://100.108.183.68:4000/health/liveliness LiteLLM liveness probe — returns 200 {"status":"healthy"} (no auth)
List routed models curl -s -H "Authorization: Bearer sk-greybox-litellm" http://100.108.183.68:4000/v1/models | python3 -c "import sys,json; [print(m['id']) for m in json.load(sys.stdin)['data']]" Show all model names LiteLLM will accept and route locally
View logs ssh greybox 'tail -40 ~/logs/litellm.log' Recent LiteLLM inference log (requests, routing, errors)
Test inference (haiku→qwen3) curl -s http://100.108.183.68:4000/v1/messages -H "x-api-key: sk-greybox-litellm" -H "anthropic-version: 2023-06-01" -H "Content-Type: application/json" -d '{"model":"claude-haiku-4-5-20251001","max_tokens":30,"messages":[{"role":"user","content":"Reply JSON: {\"ok\":true}"}]}' | python3 -c "import sys,json; r=json.load(sys.stdin); print(r['content'][0]['text'])" End-to-end inference: send Anthropic haiku request, confirm Qwen3 responds

Bot Trust & E2EE (bot-trust.ts)
Status cd ~/dev/blackbox/scripts/matrix-cross-sign && MTX_TKN=$(ssh blackbox "mcp-env.sh get COORD_MATRIX") npx tsx bot-trust.ts status Show MSK/SSK/USK presence and device signing state for all bots
Sign all devices cd ~/dev/blackbox/scripts/matrix-cross-sign && npx tsx bot-trust.ts sign-all-devices Sign all unsigned bot devices with their SSK (requires recovery key + password per bot). Run after setup-bot or if red flags appear in Element.
Sign single bot cd ~/dev/blackbox/scripts/matrix-cross-sign && npx tsx bot-trust.ts sign-all-devices --user @ig88bot:matrix.org Sign devices for one bot only
Trust users (Chris → bots) cd ~/dev/blackbox/scripts/matrix-cross-sign && npx tsx bot-trust.ts trust-users Sign each bot's MSK with Chris's USK (requires @chrislyons token)
Setup bot (new/reset) cd ~/dev/blackbox/scripts/matrix-cross-sign && npx tsx bot-trust.ts setup-bot --user @ig88bot:matrix.org Bootstrap MSK/SSK/USK cross-signing for a bot (requires browser approval). Only needed when creating a new bot or recovering lost keys — invalidates existing signatures.
