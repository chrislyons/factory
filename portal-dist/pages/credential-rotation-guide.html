Credential Flow

Architecture

All credentials live in a single age-encrypted file (secrets.env.age), decryptable by both Mac and Blackbox. As of 2026-02-26, the vault holds 18 keys — Matrix tokens migrated from plaintext files to vault. The mcp-env.sh wrapper (or systemd credentials.conf drop-in) injects specific keys into subprocess environments. Claude sessions never see secrets directly. Rotation = add-secret shortcut or manual decrypt/edit/re-encrypt/deploy.

Rotation Procedure

Universal Steps

Every credential follows this workflow: generate new value at source, update the encrypted file, deploy to Blackbox, restart affected services. The specific "generate new value" step varies per credential (see sections below).

Never leave plaintext on disk
The decrypted file exists only during editing. Shred immediately after re-encrypting.
Preferred method (single key)
~/dev/blackbox/scripts/add-secret CREDENTIAL_NAME
This decrypts, prompts securely (no echo), re-encrypts, and auto-deploys to Blackbox. Use the manual 7-step workflow below only when rotating multiple keys without an interactive session.
  1. Decrypt secrets to temp file
    age -d -i ~/.config/ig88/age.key ~/.config/ig88/secrets.env.age > /tmp/secrets.env && chmod 600 /tmp/secrets.env
  2. Generate new credential at source
    See per-credential sections below for where to generate each new value.
  3. Edit the plaintext file
    $EDITOR /tmp/secrets.env
    Replace the old value for the rotated key. Save and close.
  4. Re-encrypt
    age -e -R ~/.config/ig88/recipients.txt /tmp/secrets.env > ~/.config/ig88/secrets.env.age
  5. Shred plaintext
    rm -P /tmp/secrets.env
  6. Deploy to Blackbox
    scp ~/.config/ig88/secrets.env.age blackbox:~/.config/ig88/secrets.env.age
  7. Restart affected services
    ssh blackbox 'sudo systemctl restart matrix-coordinator'
    Then run /mcp in Claude Code to restart Mac MCP servers.

ANTHROPIC_API_KEY

Rotate Now

Controls Claude API billing. Used by the Blackbox coordinator for agent sessions and by Graphiti for entity extraction (Haiku 4.5). This is the highest-priority rotation target.

Graphiti also uses this key
After rotating, update both the age-encrypted file AND the Graphiti Docker environment. The Graphiti .env file at /home/nesbitt/projects/graphiti/.env references ANTHROPIC_API_KEY.
  1. Generate new key
    Go to console.anthropic.com/settings/keys
    Create a new key, copy it. Then revoke the old key.
  2. Update age file (steps 1-6 of generic procedure)
    Follow the universal procedure above, replacing the ANTHROPIC_API_KEY line.
  3. Update Graphiti .env on Blackbox
    ssh blackbox '$EDITOR /home/nesbitt/projects/graphiti/.env'
    Replace the ANTHROPIC_API_KEY value. Save.
  4. Restart services
    ssh blackbox 'sudo systemctl restart matrix-coordinator && cd /home/nesbitt/projects/graphiti && docker compose down && docker compose up -d'
  5. Verify
    ssh blackbox 'journalctl -u matrix-coordinator --since "1 min ago" --no-pager | head -15'
    Should show "Loaded config: 3 agents, 12 rooms" with no auth errors.

OPENROUTER_API_KEY

Rotate Soon

OpenRouter API key for zero-claude failover (BKX050). Used by OpenCode on Cloudkicker for interactive coding and by the Matrix coordinator when Anthropic API is down. Fetched on-demand via SSH from Blackbox age-encrypted secrets — never stored on disk.

  1. Generate new key
    Go to openrouter.ai/settings/keys
    Create a new key, copy it. Delete the old key.
  2. Update age file & deploy (generic procedure steps 1-6)
    Replace the OPENROUTER_API_KEY line in /tmp/secrets.env.
  3. Verify
    On Cloudkicker: oc — should launch OpenCode with working API access.
    On Blackbox: coordinator auto-failover will pick up the new key on next health check cycle.

TAILSCALE_API_KEY

No Action

Tailscale API key is ephemeral by design — generate a fresh key at time of use at tailscale.com/admin/settings/keys and use it directly. Not stored in the age vault. Old ephemeral keys expire automatically.

  1. Generate when needed, use immediately
    Go to tailscale.com/admin/settings/keys → Generate API key → use in your tool or script directly. No vault storage required — the key will expire automatically.

Self-Hosted Tokens (Graphiti, FalkorDB)

When Convenient

These tokens protect services running on your own infrastructure behind Tailscale. Lower priority since they're not exposed to the internet, but still good practice to rotate.

FALKORDB_PASSWORD is hardcoded
The FalkorDB password is currently hardcoded in /home/nesbitt/projects/graphiti/docker-compose.yml. When rotating, update both the age file AND the docker-compose file.

GRAPHITI_AUTH_TOKEN

  1. Generate new token
    openssl rand -hex 32
  2. Update three locations
    a. Age file (generic procedure)
    b. Graphiti .env: ssh blackbox '$EDITOR /home/nesbitt/projects/graphiti/.env'
    c. Deploy age file to Blackbox
  3. Restart
    ssh blackbox 'cd /home/nesbitt/projects/graphiti && docker compose down && docker compose up -d'

FALKORDB_PASSWORD

  1. Generate new password

  2. Update three locations
    a. Age file (generic procedure)

    c. Graphiti .env: ssh blackbox '$EDITOR /home/nesbitt/projects/graphiti/.env'
