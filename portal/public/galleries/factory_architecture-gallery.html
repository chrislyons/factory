Reasoning Escalation Chain

Five-Tier Routing

Five-tier escalation chain from deterministic coordinator logic up to cloud API quality. Tier 0: coordinator-rs handles all deterministic routing with no LLM. Tier 1: permanent agents (Nanbeige 3B for Boot/IG-88, Qwen 4B for Kelk) handle routine tasks. Nan acts as the triage agent for escalation decisions — routing complex tasks to Tier 2 (local deep reasoning), Tier 3 (Anthropic API), or Tier 4 (solo intensive long-context). Tier 2 can spill to Tier 3 automatically when insufficient.

Expert Subagent Architecture

Ephemeral Execution

Coordinator-rs routes tasks through autoscope (task analysis), which produces a Loop Spec defining the plan and iteration strategy. The Loop Spec dispatches to the expert subagent pool (Researcher, Coder, Admin, Policy) — all ephemeral: spawn, produce artifact, terminate. Creative subagents (Audio, Video, Music, Writing) are gated behind a human approval step. All subagents converge on an artifact output before the ephemeral instance terminates.

Autonomous Loop Integration

FCT002 §8

Autonomous loops in Agent Factory are ephemeral delegate sessions — not persistent processes. Each loop iteration spawns a Claude session with a fixed wall-clock budget (45 minutes), executes against a frozen harness with a defined mutable surface, produces a durable artifact (file, Graphiti fact, or git commit) regardless of completion state, and terminates. Every loop must pass autoscope validation before deployment: a Loop Spec defining the frozen harness, scalar metric, rollback mechanism, and approval gate must exist before the first iteration runs.

Current status: Phase 1a (Researcher loop) is ready to deploy once the first Loop Spec is produced via autoscope. Phase 1b (IG-88 Narrative loop) is blocked — narrative_accuracy_rate requires operationalization (time horizon, confidence threshold, baseline period) before a valid Loop Spec can be issued. Phase 2 (Boot Self-Improvement) is pending Phase 1 validation. Phase 3 (Research Swarm) is deferred pending Whitebox migration stability and coordinator-rs parallel dispatch implementation.

Machine Topology

Tailscale Mesh

Four-node Tailscale mesh. Whitebox (Mac Studio M1 Max) is the primary inference machine hosting all models via Ollama, plus Qdrant vector search and Graphiti knowledge graph. Blackbox (RP5, 100.87.53.109) remains the always-on coordinator — running coordinator-rs, Pantalaimon E2EE proxy, and watchdog services; Blackbox routes model API calls to Whitebox post-migration. Cloudkicker (MacBook Pro M2, 100.86.68.16) is the development workstation and delegate target for heavy compute sessions. R2D2 (iPhone) connects via Element for Matrix chat. All nodes communicate over WireGuard (Tailscale); Matrix federation runs through Blackbox to matrix.org.
