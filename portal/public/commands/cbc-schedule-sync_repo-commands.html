🔐 Gemini CLI Sandboxing Explained

The Gemini CLI is an open-source AI agent from Google/Gemini with built-in sandboxing

to limit file system access and shell command execution. This prevents the AI from accessing your entire

system.

Key Features:

Isolation: Limits file system access to project directory only

Multiple methods: macOS Seatbelt (lightweight), Docker/Podman (robust)

Easy activation: Use --sandbox flag or GEMINI_SANDBOX=true environment variable

Recommended Usage:

# Navigate to project directory

cd ~/dev/cbc-schedule-sync

# Run with sandboxing enabled

gemini --sandbox -p "Analyze the Python code in src/"

# Or enable for entire session

export GEMINI_SANDBOX=true

gemini

When sandboxing is enabled, the AI's operations are isolated to the current directory,

preventing unintended access to your host system. This is especially useful when analyzing

code or processing schedules in cbc-schedule-sync.
