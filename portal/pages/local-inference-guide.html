Foundation
Tool Stack Clarification What each tool does and how they relate REFERENCE

You're NOT conflating these—they're a layered stack:

Tool Purpose Layer Notes
llama.cpp Low-level inference engine for LLMs Engine Compiled C++ with Metal, CUDA, CPU support
MLX-LM Unified inference framework optimized for Apple Silicon Engine PRIMARY for M-series Macs: Native MLX support, superior performance on Metal, unified model library
Ollama User-friendly model server that wraps llama.cpp Server FALLBACK: For GGUF models, useful when MLX versions unavailable
LiteLLM Python library that routes API calls to any provider Router/SDK Can talk to Ollama locally, OpenRouter cloud, Anthropic API, etc.
Claude Code CLI tool for software engineering User Interface Uses LiteLLM to pick a model (local or cloud)
OpenCode Co-primary coding tool with shared memory User Interface Reads opencode.json config, also uses LiteLLM
OpenRouter Cloud API gateway (fallback when local unavailable) Cloud Provides MiniMax, Qwen, OLMo 32B when Ollama down
The Stack: LiteLLM (router) → MLX-LM (primary, M-series) → Ollama (fallback, GGUF) → OpenRouter (cloud) → actual inference
Getting Started
Start Claude Code with MLX-LM (Three-Layer Stack) mlx_lm server → LiteLLM proxy → Claude Code, fully local HOWTO
Architecture:
mlx_lm.server :8080 (OpenAI-compatible) → litellm --proxy :4000 (Anthropic↔OpenAI translator) → claude code (ANTHROPIC_BASE_URL=localhost:4000)

TL;DR — just run: launch-open-claude (MLX) or launch-ollama-code (Ollama/GGUF)
1 Install dependencies MLX-LM for inference, LiteLLM as the API translator
pip install mlx-lm
pip install 'litellm[proxy]'
2 Start mlx_lm server Launches an OpenAI-compatible inference server on port 8080
# Pick a model (all cached at ~/.cache/huggingface/hub/):
mlx_lm.server --model ~/.cache/huggingface/hub/Qwen3.5-4B-MLX-8bit --port 8080

# Verify it's running:
curl http://localhost:8080/v1/models
3 Start LiteLLM proxy Translates Anthropic Messages API → OpenAI Chat Completions, forwards to :8080
litellm --config ~/.claude/litellm.yaml --port 4000

# litellm.yaml routes all model names to localhost:8080/v1
# (mlx-qwen4b, claude-3-5-haiku-20241022, etc. → same local model)
4 Launch Claude Code Point Claude Code at LiteLLM instead of Anthropic
export ANTHROPIC_BASE_URL=http://localhost:4000
export ANTHROPIC_API_KEY=local
claude code --model claude-3-5-haiku-20241022
✅ Claude Code thinks it's talking to Anthropic. LiteLLM intercepts and routes to your local MLX model. Zero cloud, zero API key, zero quota burn.
Use the launcher scripts instead:
launch-open-claude [project-path] — MLX-LM stack (model prompt, health checks, cleanup on exit)
launch-ollama-code [project-path] — Ollama/GGUF stack (auto-registers GGUFs, same flow)
Both live in ~/dev/scripts/.
Start OpenCode Co-primary coding tool with shared memory — run launch-open-code HOWTO
1 Install OpenCode (if not already) Co-primary tool sharing memory with Claude Code
# Check if installed:
which opencode

# If not, install from https://github.com/open-code-engine/opencode
# (or use Homebrew/package manager)
2 Initialize OpenCode config Create opencode.json in your project root
{
"defaultModel": "mlx-qwen4b",
"fallbackChain": ["mlx-qwen4b", "mlx-qwen9b", "mlx-qwen2b", "openrouter"],
"providers": {
"mlx": {
"basePath": "~/.cache/huggingface/hub",
"models": [
"Qwen3.5-4B-MLX-8bit",
"Qwen3.5-9B-6bit",
"Qwen3.5-2B-MLX-8bit",
"Qwen3.5-0.8B-MLX-8bit",
"LFM2.5-1.2B-Instruct-MLX-6bit",
"LFM2.5-1.2B-Thinking-6bit",
"LFM2.5-Audio-1.5B-8bit",
"Nanbeige4.1-3B-8bit"
]
},
"openrouter": {
"apiKey": "${OPENROUTER_API_KEY}",
"models": ["allenai/olmo-3.1-32b-instruct", "qwen/qwen-32b"]
}
},
"memory": {
"path": ".opencode/memory",
"localOnly": false
}
}
3 Start OpenCode Reads opencode.json automatically
cd ~/dev/olmo
opencode
OpenCode shares the same memory directory as Claude Code via symlink at .opencode/memory → ~/.claude/projects/<slug>/memory
✅ OpenCode running with MLX-LM (OLMo 7B), shared memory with Claude Code, fallback chain: MLX → Ollama → OpenRouter

Extended Tooling in OpenCode Configure MCP servers, Qdrant, Graphiti locally HOWTO
1 Check available MCP servers List configured MCP endpoints in opencode.json
"mcp": {
"servers": {
"qdrant-mcp": {
"command": "docker run -p 6333:6333 qdrant/qdrant",
"description": "Vector database for semantic search"
},
"graphiti": {
"command": "docker run -p 8444:8444 graphiti-api",
"description": "Temporal knowledge graph"
},
"blender": {
"command": "docker run -p 8000:8000 blender-api",
"description": "3D rendering"
}
}
}
2 Start MCP servers (optional) Only needed if using specific tools like Qdrant search
# Start Qdrant vector DB:
docker run -p 6333:6333 -v qdrant_storage:/qdrant/storage qdrant/qdrant

# Start Graphiti knowledge graph:
docker run -p 8444:8444 -v graphiti_db:/data graphiti-api

# Now OpenCode tools can access these
3 Verify tools are available Check OpenCode session logs
opencode --list-tools
# or in session: /tools
MCP = Model Context Protocol. Each tool (Qdrant, Graphiti, Blender, Tailscale, etc.) runs as a separate server that OpenCode talks to.
Token Efficiency: Running all MCP servers locally is memory-intensive. Start only what you need (Ollama + Qdrant for semantic search is minimal).

New Default workflow (M-series Macs):
  1. Install MLX-LM: pip install mlx-lm
  2. Download a model: mlx_lm.generate --model mlx-community/Qwen3.5-4B-MLX-8bit
  3. Set CLAUDE_CODE_MODEL=mlx-qwen4b
  4. Start coding: claude code
  5. Optional: Keep Ollama around for legacy GGUF models via fallback chain
Architecture (clear separation):
  • MLX-LM = Optimized inference engine for Apple Silicon (PRIMARY)
  • Ollama = Server wrapper around llama.cpp for GGUF (SECONDARY/FALLBACK)
  • LiteLLM = Python router that coordinates both
  • Claude Code = CLI for coding, uses LiteLLM to select engine
  • OpenCode = Alt CLI with better MCP support, also uses LiteLLM
Troubleshooting
Common Issues & Fixes Quick solutions for startup and runtime problems REFERENCE
❌ MLX-LM model not found / "No module named 'mlx'"
# Fix: Install MLX-LM
pip install mlx-lm

# Verify installation:
python -c "import mlx; print(mlx.__version__)"

# Download model manually:
mlx_lm.generate --model mlx-community/Qwen3.5-4B-MLX-8bit --max-tokens 10
❌ "Connection refused" to MLX (LiteLLM routing issue)
# Fix: Verify LiteLLM config points to MLX
cat ~/.claude/litellm.yaml | grep mlx
# Should see your selected model in the config

# Check CLAUDE_CODE_MODEL env var:
echo $CLAUDE_CODE_MODEL
# Should be: mlx-qwen4b (or your MLX model name)
❌ Ollama fallback not working
# Fix: Ensure Ollama is running if MLX fails
ollama serve # Start in separate terminal

# Verify Ollama models are pulled:
ollama list

# Test fallback chain:
curl http://localhost:11434/api/tags # Should show models
❌ Token quota still burning even with MLX
# Fix: Verify Claude Code is using MLX, not cloud API
echo $LITELLM_CONFIG_PATH # Should point to ~/.claude/litellm.yaml
echo $CLAUDE_CODE_MODEL # Should be "mlx-qwen4b" (not anthropic-xxx)

# Check logs:
claude code --debug
❌ OOM (Out of Memory) / System freezing
# Fix: MLX 7B models use ~14GB on M-series. If freezing:

# Switch to smaller model:
export CLAUDE_CODE_MODEL=mlx-qwen4b # Qwen 4B, lighter
# OR switch to 0.8B model:
export CLAUDE_CODE_MODEL=mlx-qwen08b

# Or use OpenRouter cloud (frees local memory):
export CLAUDE_CODE_FALLBACK=true
❌ MLX model download is stuck
# Fix: Resume download (HuggingFace auto-resumes)
mlx_lm.generate --model mlx-community/Qwen3.5-4B-MLX-8bit --max-tokens 10

# Or manually check cache:
du -sh ~/.cache/huggingface/hub/

# If corrupted, delete and retry:
rm -rf ~/.cache/huggingface/hub/models--mlx-community--Qwen3.5-4B-MLX-8bit
✅ Check MLX status with: python -c "import mlx_lm; print('MLX ready')" and verify config: cat ~/.claude/litellm.yaml | grep model
Remediation Targeted fixes for specific error messages across the local inference stack REFERENCE
1 "No cookie auth credentials found" (OpenRouter 401)
# OpenCode can't reach OpenRouter — API key not injected.
# Fix: Always launch via the wrapper script:
launch-open-code # injects OPENROUTER_API_KEY from age secrets

# Or source the env manually:
source opencode-env.sh

# Verify the key is set:
echo $OPENROUTER_API_KEY | head -c 10
# Should print first 10 chars (e.g. "sk-or-v1-x")
2 "Model not found" in Ollama
# GGUF model not registered with Ollama.
# Fix: launch-ollama-code auto-registers on first run.
# Or register manually:
echo "FROM ~/.ollama/gguf/MODEL.gguf" | ollama create MODEL_NAME

# Example:
echo "FROM ~/.ollama/gguf/qwen3.5-4b-q8.gguf" | ollama create gguf-qwen4b

# Verify:
ollama list | grep MODEL_NAME
3 mlx_lm server won't start / OOM
# Model too large for available memory.
# Fix: Use a smaller model — mlx-qwen08b is only 980MB:
export CLAUDE_CODE_MODEL=mlx-qwen08b

# Close memory-heavy apps (browsers, Docker, etc.)

# Test the server directly to isolate the issue:
mlx_lm.server --model ~/.cache/huggingface/hub/Qwen3.5-0.8B-MLX-8bit --port 8080

# Check available memory:
vm_stat | head -5
4 LiteLLM 404 / "Responses API" error
# LiteLLM routing through wrong endpoint.
# Fix: Set the env var that forces chat completions URL:
export LITELLM_USE_CHAT_COMPLETIONS_URL_FOR_ANTHROPIC_MESSAGES=true

# The launch scripts (launch-ollama-code, launch-open-code) set
# this automatically. If running manually, add it to your shell profile.
5 opencode.json not found
# OpenCode doesn't see the local-mlx config.
# Fix: Copy or symlink the config to your project directory:
cp ~/dev/olmo/opencode.json ./opencode.json

# Or symlink (shares updates automatically):
ln -s ~/dev/olmo/opencode.json ./opencode.json
6 age decryption fails
# Wrong or missing identity file.
# Fix: Ensure the identity key exists:
ls -la ~/.config/ig88/age-identity.key

# If recently rotated, check BKX104 for current identity paths.
# Cloudkicker uses: ~/.ssh/id_cloudkicker_navy
# Blackbox uses: ~/.ssh/id_blackbox_navy
7 CLAUDE_CODE_MODEL not recognized
# Typo in model name. Valid names are:

# MLX models:
# mlx-qwen08b, mlx-qwen2b, mlx-qwen4b, mlx-qwen9b
# mlx-lfm-instruct, mlx-lfm-thinking, mlx-lfm-audio
# mlx-nanbeige

# GGUF models (Ollama):
# gguf-qwen08b, gguf-qwen2b, gguf-qwen4b
# gguf-lfm-instruct, gguf-lfm-thinking, gguf-lfm-audio
# gguf-nanbeige, gguf-gemma

# Verify current setting:
echo $CLAUDE_CODE_MODEL
✅ Most issues are caused by missing env vars. Use the launch scripts (launch-open-code, launch-ollama-code) to ensure correct injection.
